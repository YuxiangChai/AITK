# Translator

The customized translator is served as a bridge between the MLLM-based agents and the Android emulator, which is designed to convert the MLLM-based agents' natural language instructions into the corresponding Android emulator actions and convert the Android emulator states into the corresponding natural language instructions.

- [Customize Translator](#customize-translator)
- [Register Translator](#register-translator)

## Customize Translator

You should create a python file in the `aitk/translators` directory. The example translator is in the `aitk/translators/qwen25_vl.py`.

The translator class should inherit from the `BaseTranslator` class in the `aitk/translators/base.py` and should contain the following functions:

- `to_device(self, action: str, width: int, height: int) -> dict`: convert the string generated by agent to the action that can be executed on the device
- `to_agent(self, task: str, state: dict, history: dict) -> str`: convert the Android emulator information into the corresponding natural language instructions and return the agent's response string

### to_device

The `to_device` function should return an action dictionary in the following action space:

```json
{"action": "open", "app": "youtube"}
{"action": "tap", "x": 100, "y": 100}
{"action": "long_press", "x": 100, "y": 100, "duration": 1000 (optional, default to 1000ms)}
{"action": "swipe", "x1": 100, "y1": 100, "x2": 200, "y2": 200, "duration": 1000 (optional, default to 1000ms)}
{"action": "type", "text": "hello"}
{"action": "enter"}
{"action": "back"}
{"action": "home"}
{"action": "wait", "time": 1 (optional, default to 3 second)}
{"action": "end", "answer": "answer to the user query (default to empty string)"}
```

The input arguments of the `to_device` function are:

- `action` (str): the response string generated by the agent
- `width` (int): the width of the screenshot
- `height` (int): the height of the screenshot

The output is a dictionary containing the action that can be executed on the device.

### to_agent

The `to_agent` function should return a string of natural language instructions for the agent.

The input arguments of the `to_agent` function are:

- `task` (str): the task description
- `state` (dict): the current state of the Android emulator
  - `xml`: the XML of the current state of the device in string format
  - `screenshot`: the screenshot of the current state of the device in base64 format
  - `package`: the package of the current app
  - `activity`: the activity of the current app
- `history` (dict): the history of the task execution
  - `xmls`: a list of XMLs of the device in the previous steps
  - `screenshots`: a list of screenshots of the device in the previous steps
  - `actions`: a list of actions of the device in the previous steps
  - `agent_messages`: a list of messages from the agent in the previous steps
  - `packages`: a list of packages of the apps in the previous steps
  - `activities`: a list of activities of the apps in the previous steps

The output is a response string from the agent, which will be passed to the `to_device` function as the `action` argument.

## Register Translator

You should modify the `configs/controller.yaml` file to set the translator class.
